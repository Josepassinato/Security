# ISO/IEC 27001 90-day roadmap

## Days 1-15: scope and gap assessment

- Confirm certification scope and exclusions.
- Appoint ISMS owner and backup owner.
- Inventory assets: repositories, servers, domains, secrets, providers, customer-facing systems.
- Review current access paths: SSH, GitHub, PM2/VPS, DNS, email, LLM providers, payment/CRM if used.
- Run initial risk assessment.
- Create gap report against ISO/IEC 27001 clauses and Annex A controls.

Acceptance evidence:

- Signed scope statement.
- Asset inventory v1.
- Risk register v1.
- Gap assessment v1.

## Days 16-45: policies and core controls

- Approve security policy.
- Approve risk assessment and risk treatment methodology.
- Define access control policy and access review cadence.
- Define backup and restore policy.
- Define secure development and change management policy.
- Define incident response policy.
- Define supplier security policy.
- Define evidence retention policy.
- Enable or document MFA for critical systems.
- Document production deployment flow.
- Document secrets handling and key rotation process.

Acceptance evidence:

- Policy set approved.
- Access review completed.
- Backup test completed.
- Change management sample with ticket/commit/deploy evidence.
- Incident response tabletop record.

## Days 46-70: operate and collect evidence

- Run weekly access review for critical systems.
- Run dependency/vulnerability review.
- Collect deployment records.
- Collect logs and monitoring records.
- Create supplier register.
- Create security training record for operators.
- Run restore simulation.
- Run incident tabletop simulation.
- Complete Statement of Applicability v1.

Acceptance evidence:

- Evidence register populated.
- SoA v1 approved.
- Risk treatment plan updated.
- Management metrics v1.

## Days 71-85: internal audit and management review

- Run internal audit against scope.
- Record nonconformities and observations.
- Correct critical gaps.
- Hold management review.
- Approve external certification readiness decision.

Acceptance evidence:

- Internal audit report.
- Corrective action log.
- Management review minutes.
- Certification body shortlist.

## Days 86-90: external audit preparation

- Select certification body.
- Package evidence by control.
- Prepare system description and architecture.
- Prepare staff for interviews.
- Confirm public wording: "readiness program" until certificate is issued.

Acceptance evidence:

- Certification body engagement started.
- Audit evidence room ready.
- External audit schedule proposed.
