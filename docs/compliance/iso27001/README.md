# ISO/IEC 27001 readiness package

Status: preparation baseline. This folder does not claim certification. It defines the first certification-ready operating package for the 12Brain / Quarry information security management system.

## Certification target

Target standard: ISO/IEC 27001:2022.

Recommended first scope:

> Information Security Management System for the development, operation, support, and commercial delivery of Quarry, including source code, production infrastructure, access management, customer-facing evidence, supplier controls, incident response, backup, change management, and security monitoring.

The first scope is intentionally narrow. It covers the product that needs market credibility without dragging every unrelated 12Brain product into the first audit.

## Folder contents

- `scope.md` — proposed ISMS scope, boundaries, exclusions, and dependencies.
- `roadmap-90-days.md` — execution plan from gap assessment to external audit readiness.
- `risk-register.md` — initial risk register for Quarry operations.
- `statement-of-applicability.md` — initial Annex A applicability map.
- `evidence-register.md` — evidence library the auditor will expect.
- `policies-index.md` — minimum policy set to operate the ISMS.

## Operating rule

No document in this folder is proof by itself. Every control needs evidence: ticket, log, screenshot, report, access review, test result, incident record, backup verification, or management review.

## Public wording

Until certification is issued by an accredited certification body, public materials must use:

- "ISO/IEC 27001 readiness program"
- "ISO/IEC 27001-aligned controls"
- "certification in preparation"

Public materials must not use:

- "ISO 27001 certified"
- "ISO 27001 compliant"
- "audited under ISO 27001"

Those claims are only allowed after successful certification and review of the certificate scope.
